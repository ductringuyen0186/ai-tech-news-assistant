## Goal

Rebuild the TechPulse AI frontend to a **Vercel/Linear-style developer
dashboard** — dark default, dense information, sharp typography, sidebar
navigation, polished motion, and three new engagement features
(suggested follow-up questions, saved research, citation hover cards).

The immediate trigger: a screenshot showing the Research agent's
markdown table rendering as raw pipes and dashes — the M3 inline
markdown renderer doesn't handle tables. That bug is one item in a
broader polish gap across all 6 tabs. This mission addresses both.

## Why now

Mission 2 just landed the multi-agent architecture and proved it works
end-to-end. The architecture is recruiter-worthy. The UI isn't. The
gap between "what's under the hood" and "what shows on screen" is
currently the bottleneck on portfolio impact. A research recruiter
clicking into the app today sees a basic shadcn/ui demo with a broken
markdown table. After this mission, they should see something that
looks like an engineering tool they'd actually use.

## Users

- **Primary**: backend / infra / platform engineering recruiters
  scanning the app for ~2-5 minutes during portfolio review. They
  expect Vercel-grade polish from a senior IC. First impression decides
  whether they read the code.
- **Secondary**: the maintainer using the app as a real news-research
  tool. Power features (saved research, Cmd+K palette, keyboard shortcuts)
  should make daily use snappy.

## Scope

### In scope (locked from grill round 1-3)

- **All 6 tabs polished**: News Feed, Research, Knowledge Graph, Digest,
  Ask AI, Settings — every visible surface
- **Desktop only** (~1280-1920px). Mobile is explicitly out of scope —
  no responsive breakpoints, no touch optimization
- **Visual style**: developer-dashboard (Vercel/Linear vibe). Dark
  default + light option. Single accent color (blue). Status palette
  for done/running/error
- **Navigation**: left sidebar with 6 nav items + global Cmd+K command
  palette for power navigation
- **Density**: Linear-dense (13-14px body text, 4-8px row padding,
  more content per scroll)
- **Empty states**:
  - Research tab: 4-6 high-quality suggested query chips
  - News Feed tab: "Trending Now" rail with top topics from corpus
- **Markdown rendering**: replace the M3 inline renderer with
  `react-markdown + remark-gfm`. Fixes tables, code blocks, footnotes,
  task lists, strikethrough
- **Engagement feature 1 — Suggested follow-ups**: after a research
  report renders, surface 3 follow-up questions the agent thinks the
  user might ask next. Click to run as a fresh query
- **Engagement feature 2 — Saved research**: new backend `saved_research`
  table (SQLite) with `(id, question, report_md, sources_json,
  created_at)`. REST endpoints `POST/GET/GET-by-id/DELETE`. New "Saved"
  sidebar entry. Save button on completed research reports
- **Engagement feature 3 — Citation hover cards**: hovering on any
  `[N]` citation pops a card with title + source + summary preview +
  publish date. Click jumps to source as before
- **Motion**: polished with `framer-motion`. Animate subagent row
  enter/exit, panel collapse, citation hover-card pop, page-tab
  transitions, suggested-query stagger
- **Validation**: tightened Playwright suite — each tab gets a
  happy-path test PLUS a final-state screenshot baseline visually
  approved once. Catches regressions and missed polish

### Out of scope (explicit non-goals)

- Mobile / tablet responsive design — desktop-first, single-breakpoint
- Touch interactions, haptics, native-app feel
- New tabs beyond "Saved" (Saved Research). NO new top-level surfaces
  like Trending or Source Comparison
- Server-side rendering, edge functions, hydration optimization
- A11y push beyond the basics (focus rings, keyboard nav for the new
  command palette, semantic landmarks). WCAG AA full audit is deferred
- Browser support beyond modern Chrome / Firefox / Safari (latest 2
  versions)
- Logo redesign or rebrand — TechPulse AI name and current logo stay
- Performance budget tracking with Lighthouse CI — deferred to v2
- Internationalization / RTL support
- Editing or annotating reports (read-only)
- Sharing research reports via URL — defer to v2

## Success criteria

1. The reported markdown table bug is fixed — agent reports with
   `| Company | Amount |` style tables render as proper HTML tables
   with borders and alignment
2. Recruiter walking through the app sees consistent visual language
   across all 6 tabs: same sidebar, same typography, same color
   accents, same hover/focus patterns
3. Dark mode works on every tab; theme persists in localStorage; light
   mode is the secondary option but fully functional
4. Cmd+K palette opens from anywhere; lists all tabs + recent
   research; arrow keys + Enter navigate; Esc closes
5. Empty Research tab shows 4-6 suggested query chips that, when
   clicked, populate the input and submit
6. Completed research report shows 3 suggested follow-up questions
   below the report; clicking one fires a fresh research run
7. Save button on a research report writes to backend; "Saved" sidebar
   tab lists prior reports; clicking one renders the saved report
8. Hovering any `[N]` citation in any rendered markdown shows a
   small card (~300px wide) with article title + source + summary
   preview within 200ms
9. Subagents panel rows enter/exit with subtle framer-motion stagger
   (no jarring snap-in)
10. New Playwright per-tab smoke catches the table-rendering fix AND
    visually approves the final state of every tab
11. 35 existing Playwright tests still green; backend M2 contract
    tests still green; live research test still 3/3

## Constraints

- **Tech stack frozen**: React + Vite + TypeScript + shadcn/ui +
  Tailwind. No framework swap
- **New dependencies allowed** (scope-justified):
  - `react-markdown` + `remark-gfm` for markdown rendering
  - `framer-motion` for animations (~20KB acceptable on desktop-only)
- **Backend**: SQLite `saved_research` table + REST endpoints. No new
  ORM, no migration framework — match the existing schema-creation
  pattern in `database/init_db.py`
- **Compatibility**: existing 6 Playwright tests in `research.spec.ts`
  must stay green (the M5 subagents-panel test is included). The
  backend's M2 contract tests stay green. Live research tests stay
  green.
- **Time budget**: ~7 days (one focused week). 6 milestones
- **Don't break**: existing keyboard shortcuts (none currently), URL
  routing (currently SPA-only, tab state in component state — keep
  that)

## Data

- **Inputs**: existing 80-article corpus, existing entity tables,
  existing research output. NO new data sources
- **New persistence**: `saved_research` table — `id INTEGER PRIMARY
  KEY`, `question TEXT`, `report_md TEXT`, `sources_json TEXT`,

  density preference if shipped, recent-research IDs cache for
  Cmd+K (~last 20 IDs)

## Failure modes & handling

| Risk | Mitigation |
|------|------------|
| `react-markdown` strips required HTML (e.g. citation classes) | Use `remark-gfm` for GFM + a small remark plugin to preserve `[N]` → `<a class="citation">` rewrite during AST walk. NO `rehype-raw` (security risk). Test on the failing table screenshot's exact input |
| framer-motion bundle size impacts cold load | Only desktop; ~20KB gzipped acceptable. Tree-shake (import only `motion`, `AnimatePresence`). Verify in Vite build report |
| Cmd+K palette conflicts with browser shortcuts | `Cmd+K` on macOS, `Ctrl+K` on Windows. Use a robust hotkey lib pattern (matches DOM events; preventDefault). Test in DevTools |
| Sidebar nav rewrite breaks existing Playwright | The 35 existing tests use `getByRole("tab", ...)` — rewrite sidebar to use semantically equivalent `role="tab"` items in a `role="tablist"` so selectors stay valid. Document this convention |
| Saved research backend table migration breaks fresh DB initialization | Add an "IF NOT EXISTS" guard. Test against a fresh `news.db` |
| Hover cards spam the backend with /api/news/{id} requests | Debounce 200ms before fetch; cache fetched articles in a `Map<id, Article>` for the session |
| Theme flash on page load (FOUC) | Inline a `<script>` in `index.html` that reads `localStorage.techpulse-theme` and sets `<html class="dark">` BEFORE React mounts. Standard pattern |
| Per-tab visual baselines flake on font rendering | Use Playwright's `toHaveScreenshot` with reasonable threshold (0.1). Lock fonts in CSS to system stack so OS differences don't matter |

## Alternatives considered (and rejected)

- **Mobile-first / responsive** — rejected: user explicitly chose
  "Desktop only" in grill round 1. Saves significant design + test
  matrix
- **Top tabs (current, polished)** — rejected: user picked sidebar.
  Sidebar earns the dashboard credibility worth more than the smaller
  blast radius
- **Per-tab accent color** — rejected: user picked single accent.
  Visual coherence > distinctiveness for this audience
- **localStorage-only saved research** — rejected: user picked
  backend table. Durability > simplicity given the portfolio context
- **Skip Cmd+K** — rejected: it's the single feature that signals
  "this person ships polished tools." Worth the 200 lines
- **Extend inline markdown renderer in place** — rejected: user
  picked react-markdown. Library is mature; the inline renderer was a
  pragmatic v1 that's now blocking real outputs (tables, code blocks)
- **Live demo video as validation only** — rejected: the per-tab
  Playwright smoke catches regressions. Video is nice-to-have, not
  primary validation
